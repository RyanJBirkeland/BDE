# Archived Specs

Specs that have been either **SHIPPED**, **ABANDONED**, or **SUPERSEDED**. Each file carries a `> **Status: ...**` banner at the top explaining its current state (verified 2026-04-21).

## Why keep them?

- Audit trail: the decisions that shaped the current code.
- Context for readers who encounter references to the spec elsewhere (commits, review comments, etc.).
- Reference material when a similar problem resurfaces.

## What's NOT here

Live work — specs that are still in progress, PARTIAL, or UNCLEAR — stays in `docs/specs/`.

## Renaming convention

Top-level specs moved here keep their original filenames. The series specs (AX, DP, FC, TF, TQ, LD) retain their original `<series>-s<n>-<slug>.md` names so cross-references in epic docs continue to work via simple path adjustment (`docs/specs/` → `docs/archive/specs/`).
